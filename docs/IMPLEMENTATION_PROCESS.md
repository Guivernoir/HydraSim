# HydraSim Implementation Process

HydraSim uses a conservative implementation process for simulator features.

## 1. Lock

Before adding behavior, document:

- what the slice enables;
- what it does not enable;
- required input contracts;
- expected outputs;
- safety boundaries;
- test evidence.

## 2. Implement

Implementation should be narrow and deterministic:

- no hidden network behavior;
- no broad external targets;
- no operational-data claims;
- no dependency expansion unless the slice explicitly needs it;
- optional dependencies must fail cleanly when absent.

## 3. Verify

Every slice should run:

```bash
.venv/bin/python -m pip install -e ".[dev,modbus]"
.venv/bin/python tools/check_project_quality.py
.venv/bin/python -m black --check src/wt_simulator/scenarios src/wt_simulator/ics src/wt_simulator/hydraulics tests tools
.venv/bin/python -m unittest discover -s tests -v
.venv/bin/python -m compileall -q src tests
find src tests -type d -name __pycache__ -prune -exec rm -rf {} +
git diff --check
```

The full gate must include the live `pymodbus` Modbus end-to-end tests. Do not
count a slice as verified if those tests are skipped because `pymodbus` is not
installed. Generated bytecode and caches must be removed before handoff.

## 4. Handoff

The handoff should state:

- what changed;
- what commands were run;
- which optional checks were skipped and why;
- which slice is next.
