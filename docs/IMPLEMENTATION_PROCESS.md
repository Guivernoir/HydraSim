# HydraSim Implementation Process

HydraSim uses a conservative implementation process for simulator features.

## 1. Lock

Before adding behavior, document:

- what the slice enables;
- what it does not enable;
- required input contracts;
- expected outputs;
- safety boundaries;
- test evidence.

## 2. Implement

Implementation should be narrow and deterministic:

- no hidden network behavior;
- no broad external targets;
- no operational-data claims;
- no dependency expansion unless the slice explicitly needs it;
- optional dependencies must fail cleanly when absent.

## 3. Verify

Every slice should run:

```bash
python -m black --check <touched-python-paths>
python -m unittest discover -s tests -v
python -m compileall -q src tests
git diff --check
```

Generated bytecode and caches must be removed before handoff.

## 4. Handoff

The handoff should state:

- what changed;
- what commands were run;
- which optional checks were skipped and why;
- which slice is next.
